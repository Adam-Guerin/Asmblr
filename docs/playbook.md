# Venture Playbook (Local RAG)

Use this playbook to keep outputs grounded and consistent.

## Principles
- Prioritize clear pains with observable signals.
- Favor MVPs that are buildable in weeks, not months.
- Avoid dependencies on paid APIs.
- If evidence is weak, mark assumptions explicitly.

## Scoring Heuristics
- Market size: proxy via number of mentions, buyer type, urgency.
- Competition: count obvious incumbents and ease of differentiation.
- Build difficulty: integrations, data sources, auth, compliance.
- Time to MVP: 2-6 weeks ideal.

## Output Quality
- Use clear headings.
- Provide short, actionable bullet points.
- Always include source URLs used.
